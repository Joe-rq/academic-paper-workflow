# 评审产出模板

> 本文件固化 paper-review-team 的**产出格式**，确保 4 个独立 teammate 与 lead 汇总结构统一。
> 评审**方法论**（4 角色定义、7 维度检查项、决策逻辑）见 `.claude/skills/paper-workflow/references/peer-review-simulation.md`——本文件只规定"评审产出长什么样"，不重复方法论。

---

## 一、单角色评审表（每位 teammate 填一份）

写到各自的 `output/reviews/{role}.md`（role = `eic` / `method` / `domain` / `devil` / `statistician` / `reproducibility` / `ethicist`，后三个按 `reviewer_set.roles` 按需），直接套用：

```md
# 评审意见 — {角色中文名}

- **评审对象**：draft/paper.md
- **评审角色**：{主编 / 方法论专家 / 领域专家 / 魔鬼代言人}
- **日期**：{YYYY-MM-DD}

## 总体结论
- 建议：[接受 / 小修 / 大修 / 拒绝]
- 创新性：[1-5]　方法严谨性：[1-5]　写作质量：[1-5]
- 一句话总评：……

## 7 维度诊断
| 维度 | 得分(1-5) | 最关键的问题 |
|------|-----------|-------------|
| 选题与创新性 |  |  |
| 文献综述 |  |  |
| 研究方法 |  |  |
| 结果与讨论 |  |  |
| 论证逻辑 |  |  |
| 写作质量 |  |  |
| AI 痕迹 |  |  |

> 各角色按 peer-review-simulation.md 的关注重点深挖相应维度，不必每项长篇——突出本角色视角最该抓的问题。

## 主要优点
1. ……
2. ……

## 主要问题（按严重程度排序）
1. **[高]** ……（定位到具体章节/段落）
2. **[中]** ……
3. **[低]** ……

## 修改建议
1. ……
```

**魔鬼代言人特别注意**：可以不写优点；问题必须尖锐、具体、可操作，给出"这会让论文被拒"的真实理由，不要客套。这是本角色存在的意义。

## 二、汇总报告（lead 产出）

lead 读完全部 4 份评审后，汇总写到 `output/review-report.md`：

```md
# 同行评审汇总报告

- **评审对象**：draft/paper.md
- **日期**：{YYYY-MM-DD}
- **评审人**：N（按 `reviewer_set.roles`，基础 4 + 按需专业角色）

## 1. 总体结论
- 综合建议：[接受 / 小修 / 大修 / 拒绝]
- 4 人结论分布：接受 N / 小修 N / 大修 N / 拒绝 N
- 均分：创新性 X · 方法 X · 写作 X

## 2. 共识问题（≥2 位评审人提及）
### P1 必须改（高严重度共识）
| # | 问题 | 共识人数 | 修改方向 |
|---|------|---------|---------|
| 1 | …… | 3/4 | …… |

### P2 应该改（中严重度共识）
……

## 交叉验证硬伤（多角色独立命中，最高置信）

> 同一问题被 ≥2 个互不可见独立上下文的角色命中 = 高置信硬伤（方法见 `conflict-detection.md`）。优先级高于普通共识，Worker / Verifier 验收时最该盯。单角色命中（即便尖锐）不入此表，归 §2 共识或单列。

| # | 问题 | 命中角色 | 各自证据（原文 / 重算片段） | 置信度 |
|---|------|---------|---------------------------|--------|
| 1 | …… | devil, statistician | devil: "……"；statistician: "重算 p=……" | 🔴 高 |

## 3. 分歧问题（评审人意见不一）
| # | 观点 A | 观点 B | 采纳 | 理由 |
|---|--------|--------|------|------|
| 1 | …… | …… | A | …… |

## 4. 魔鬼代言人专项
- 最尖锐的拒稿风险：……
- 是否成立：……
- 应对：……

## 5. 修改路线图
| 优先级 | 问题 | 涉及章节 | 预估工作量 |
|--------|------|---------|-----------|
| P1 | …… | …… | …… |
| P2 | …… | …… | …… |
| P3 | …… | …… | …… |

## 6. 下一步
- [ ] P1 全部修改完毕后重新评审，或
- [ ] 已达投稿标准 → 进入 Stage 8（revise）/ Stage 9（build）
```

## 三、决策参考

汇总后按 peer-review-simulation.md Phase 3 的决策标准判断去向：

| 评审结果 | 去向 |
|---------|------|
| 4 位均通过 | 直接 Stage 8 |
| 3 通过 / 1 保留 | 针对性修改 |
| 2 通过 / 2 保留 | 重点改后重新评审 |
| ≤1 通过 | 回 Stage 4 重审框架 |

本模板只提供产出骨架，不重复该决策逻辑。
