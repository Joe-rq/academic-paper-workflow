# 同行评审模拟指南

> 替代 academic-paper-reviewer skill 的内置评审方法论。适用于 Stage 7。

---

## 评审团队（基础 4 角色 + 按需专业角色）

基础 4 角色覆盖通用论文评审；按需专业角色在 `paper-review-team` 多 agent 模式下根据论文类型动态加入（组队规则见 `../paper-review-team/SKILL.md`）。单 agent 串行模拟（Stage 7 默认）通常只跑基础 4 角色。

### 评审人 A：主编（EIC）
**关注**：论文整体质量、创新性、与期刊匹配度
**核心问题**：这篇论文是否值得发表？

### 评审人 B：方法论专家
**关注**：研究设计、数据收集、分析方法
**核心问题**：研究方法是否严谨？

### 评审人 C：领域专家
**关注**：理论贡献、实践意义、文献引用
**核心问题**：研究对该领域的贡献是什么？

### 评审人 D：魔鬼代言人
**关注**：主动寻找论文的弱点和漏洞
**核心问题**：什么能让这篇论文被拒？

### 按需角色（paper-review-team 多 agent 模式按论文类型组队时加入）

以下角色仅在 `paper-review-team` 检测到对应论文特征时 spawn（单 agent 串行模拟不跑）。各角色标注维度重点，供深挖相应维度。

### 评审人 E：统计审稿人（statistician）— 定量 / RCT / 计量论文按需
**关注**：统计前提、效应量、样本量与功效、模型选择、多重比较校正、缺失数据处理偏倚
**核心问题**：统计结论是否经得起重算？（范例：主终点置信区间与报告的 p 值不自洽，按原文数据重算 p 应远小于报告值）
**维度重点**：研究方法、结果与讨论

### 评审人 F：复现审稿人（reproducibility）— 含代码 / 数据论文按需
**关注**：代码与数据开放、实验可复跑、基准公平性、超参与随机种子、结果稳定性
**核心问题**：换个人换台机器，结论还成立吗？
**维度重点**：研究方法、结果与讨论

### 评审人 G：伦理审稿人（ethicist）— 涉人体 / 动物研究按需
**关注**：IRB / 伦理委员会批件、知情同意、数据保护、利益冲突披露、试验注册号、动物实验与双重用途
**核心问题**：研究在伦理与合规上是否无懈可击？（这类风险其他角色不会重点检查）
**维度重点**：研究方法

## 评审维度（7 维）

### 1. 选题与创新性

| 检查项 | 通过标准 | 常见问题 |
|--------|---------|---------|
| 研究问题是否明确？ | 能用一句话说清 | 问题过于宽泛 |
| 是否有创新点？ | 至少 1 个明确的创新 | 只是重复已有研究 |
| 研究价值如何？ | 有理论或实践意义 | 纯学术游戏 |

### 2. 文献综述

| 检查项 | 通过标准 | 常见问题 |
|--------|---------|---------|
| 覆盖面是否足够？ | 核心文献不遗漏 | 遗漏重要文献 |
| 是否按主题分类？ | 不是逐篇列举 | 流水账式综述 |
| 是否指出研究缺口？ | 明确的缺口陈述 | 只有综述无缺口 |

### 3. 研究方法

| 检查项 | 通过标准 | 常见问题 |
|--------|---------|---------|
| 方法是否适合研究问题？ | 方法-问题匹配 | 方法与问题脱节 |
| 数据收集是否规范？ | 来源明确、过程透明 | 数据来源不清 |
| 分析方法是否恰当？ | 分析工具合理 | 方法使用错误 |

### 4. 结果与讨论

| 检查项 | 通过标准 | 常见问题 |
|--------|---------|---------|
| 结果是否回答了研究问题？ | 一一对应 | 结果与问题脱节 |
| 是否有具体数据支撑？ | 量化+质性证据 | 只有定性描述 |
| 讨论是否深入？ | 超越重复结果 | 只重述结果 |

### 5. 论证逻辑

| 检查项 | 通过标准 | 常见问题 |
|--------|---------|---------|
| 论点是否清晰？ | 每个论点可识别 | 论点模糊 |
| 论据是否充分？ | 每个论点有 ≥1 个证据 | 有论点无论据 |
| 是否有逻辑跳跃？ | 推理链条完整 | 结论与证据不匹配 |

### 6. 写作质量

| 检查项 | 通过标准 | 常见问题 |
|--------|---------|---------|
| 语言是否学术化？ | 无口语化表达 | 口语化、模糊 |
| 结构是否清晰？ | 段落有过渡、层次分明 | 结构混乱 |
| 引用是否规范？ | GB/T 7714 格式统一 | 格式不统一 |

### 7. AI 痕迹

| 检查项 | 通过标准 | 常见问题 |
|--------|---------|---------|
| 是否有 AI 高频词？ | 无"此外""值得一提的是"等 | AI 味明显 |
| 句式是否单调？ | 长短句交替、有变化 | 并列对称堆叠 |
| 数据呈现是否多样化？ | 不全是"X%的学生……" | 格式单一 |

## 评审流程

### Phase 1：独立评审

每位评审人独立阅读论文，填写评审表：

```
评审人：[角色]
总体评价：[接受/小修/大修/拒绝]
创新性：1-5
方法严谨性：1-5
写作质量：1-5
主要优点：
  1. ...
  2. ...
主要问题：
  1. [严重程度：高/中/低] ...
  2. ...
修改建议：
  1. ...
```

### Phase 2：汇总分析

1. 收集评审人意见（单 agent 模式为基础 4 位；`paper-review-team` 多 agent 模式按组队角色集）
2. 按严重程度排序所有问题
3. 标记共识问题（≥2 位评审人提到）
4. 标记分歧问题（评审人意见不一致）

> **深层交叉验证（仅 paper-review-team 多 agent 模式启用）**：Phase 2 只做浅层的共识 / 分歧标记。`paper-review-team` 在此之上做显式交叉验证 + 冲突裁决——同一问题被 ≥2 个互不可见独立上下文的角色命中视为高置信硬伤，并对评价相反 / 建议矛盾 / 优先级冲突做分级裁决。方法见 `../paper-review-team/references/conflict-detection.md`。单 agent 串行模拟不具备真独立上下文，不启用此层。

### Phase 3：决策与修改路线图

**决策标准**：

| 评审结果 | 行动 |
|---------|------|
| 4 位均通过 | 直接进入 Stage 8 |
| 3 位通过，1 位有保留 | 针对性修改 |
| 2 位通过，2 位有保留 | 重点修改后重新评审 |
| ≤1 位通过 | 回到 Stage 4 重新审视框架 |

**修改路线图**：
```
优先级 1（必须修改）：共识问题中的高严重度项
优先级 2（应该修改）：共识问题中的中严重度项
优先级 3（建议修改）：个别评审人提出的改善建议
```

## 常见拒稿原因速查

| 原因 | 症状 | 预防措施 |
|------|------|---------|
| 创新性不足 | 与已有研究高度相似 | 文献矩阵对比，明确差异化 |
| 方法论缺陷 | 样本量不足/方法不当 | 预先设计方法，请教领域专家 |
| 数据不支撑结论 | 结论超出数据范围 | 每个结论标注支撑数据 |
| 结构混乱 | 评审人无法跟上逻辑 | 大纲阶段做好论点递进 |
| 语言问题 | 口语化或 AI 痕迹 | 使用 ai-pattern-detection.md 检查 |
