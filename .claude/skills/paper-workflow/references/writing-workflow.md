# 论文写作工作流：6 阶段方法论

> 从选题到定稿的完整写作流程。每个阶段有明确的输入、输出和质量门控。

---

## 全局视图

```
选题定位 → 框架搭建 → 内容填充 → 学术规范 → 质量验收 → 定稿输出
  (1-2天)    (1-2天)    (3-5天)    (1-2天)    (1-2天)     (1天)
```

---

## 阶段 1：选题定位

**目标**：确定研究问题与价值

### 输入
- 用户的研究兴趣或原始材料
- 初步文献搜索结果

### 关键任务
1. 使用头脑风暴框架（`brainstorming-guide.md`）明确研究问题
2. 初步文献检索（关键词搜索，筛选 10–15 篇核心文献）
3. 确定目标期刊（至少 2 个候选）
4. 评估可行性（数据获取、方法掌握、时间预算）

### 输出
- 研究问题陈述（1 段话）
- 目标期刊清单（含投稿要求摘要）
- 核心文献列表（10–15 篇）

### 质量门控
- [ ] 研究问题能用一句话说清吗？
- [ ] 研究问题有明确的理论或实践价值吗？
- [ ] 数据来源确定了吗？

---

## 阶段 2：框架搭建

**目标**：构建论文骨架与论点体系

### 输入
- 阶段 1 的所有产出
- 完整的文献综述初稿

### 关键任务
1. 确定论文的 6 大部分及二级标题
2. 为每章写核心论点（每章 1 句话）
3. 规划图表清单
4. 分配字数预算

### 论文骨架模板

```markdown
# 论文标题

## 摘要（200-300字）
## 关键词（3-5个）

## 一、引言（800-1200字）
### （一）研究背景与问题提出
### （二）研究目的与意义
### （三）核心概念界定

## 二、文献综述（1500-2500字）
### （一）[类别A]研究
### （二）[类别B]研究
### （三）[类别C]研究
### （四）研究述评与本文定位

## 三、研究方法（1000-1500字）
### （一）研究设计
### （二）研究对象
### （三）研究工具
### （四）研究程序
### （五）数据分析方法

## 四、研究结果（2000-3000字）
### （一）研究情境
### （二）核心发现
### （三）深入分析

## 五、讨论（1000-1500字）
### （一）研究问题的回答
### （二）理论贡献
### （三）实践意义
### （四）研究局限
### （五）未来研究方向

## 六、结论（500-800字）

## 参考文献
```

### 输出
- 完整的论文大纲（到二级标题 + 每节核心论点）
- 字数分配表
- 图表清单

### 质量门控
- [ ] 每章的核心论点能用 1 句话说清吗？
- [ ] 章节之间有清晰的逻辑递进吗？
- [ ] 字数总和在目标期刊要求范围内吗？

---

## 阶段 3：内容填充

**目标**：用案例、数据、论证充实论文骨架

### 输入
- 阶段 2 的论文大纲
- 研究数据（原始数据、分析结果）
- 文献笔记

### 关键任务

#### 按章节推进（推荐顺序：方法→结果→讨论→文献综述→引言→摘要）

1. **方法**：按 5-Move 结构写作，确保可复现性
2. **结果**：先做图表，再写文字解读
3. **讨论**：从结果出发，上升到理论和实践层面
4. **文献综述**：按 3-Move 结构组织，先分类再找局限
5. **引言**：最后写，确保与全文呼应
6. **摘要**：全文完成后写

#### 证据标注
- 使用证据分层体系（`evidence-labeling.md`）标注所有信息
- 初稿阶段 T3/T4 可大量使用
- 每完成一章，将 T3/T4 推进为 T1/T2

### 输出
- 论文初稿（所有章节完成）
- 证据标注状态汇总

### 质量门控
- [ ] 所有章节完成了吗？
- [ ] 每个论点都有证据支撑吗？
- [ ] 图表全部制作并嵌入了吗？

---

## 阶段 4：学术规范

**目标**：语言、格式、引用规范化

### 输入
- 论文初稿

### 关键任务
1. **引用格式统一**：按目标期刊要求（GB/T 7714 / APA / 芝加哥等）
2. **统计符号检查**：斜体、空格、精确度
3. **格式统一**：标题层级、编号体系、图表格式
4. **语言润色**：消除口语化表达、统一术语
5. **清理标注**：所有 T3/T4 推进为 T1/T2 或删除
6. **降 AIGC**：检查并列排比、对称句式等 AI 特征

### 常见规范问题速查
- 参考文献中英文作者姓全大写
- 半角/全角标点统一
- 图表编号连续
- 页码、DOI 信息完整
- 基金项目标注规范

### 输出
- 规范化稿

### 质量门控
- [ ] 引用格式全文统一吗？
- [ ] 统计符号格式正确吗？
- [ ] 还有 T3/T4 标注残留吗？

---

## 阶段 5：质量验收

**目标**：5 维度检查与修改

### 输入
- 规范化稿

### 关键任务
1. 使用质量检查清单（`quality-checklist.md`）逐项检查
2. 完成自评表，得分 ≥ 7.0 方可进入下一阶段
3. 针对低分维度集中修改
4. 同行/导师审阅（如有条件）

### 输出
- 终稿（经质量验收通过）
- 自评表

### 质量门控
- [ ] 5 维度全部通过了吗？
- [ ] 自评表得分 ≥ 7.0 了吗？
- [ ] 常见问题速查无遗漏吗？

---

## 阶段 6：定稿输出

**目标**：格式转换与存档

### 输入
- 终稿（Markdown 格式）

### 关键任务
1. 转换为目标格式（Word / PDF / LaTeX）
2. 检查转换后格式是否正确（图表、公式、引用）
3. 双语摘要最终校对
4. 作者信息、基金项目、致谢等元数据补充
5. 存档（含版本号）

### 输出
- 投稿格式文件（Word/PDF）
- 存档版本（Markdown 原稿 + 投稿格式）

### 质量门控
- [ ] 转换后格式与原稿一致吗？
- [ ] 元数据（作者、基金等）完整吗？
- [ ] 存档文件命名规范吗？

## 快速决策卡

遇到以下情况时，参考对应的决策路径：

| 情况 | 决策 |
|------|------|
| 不确定研究问题是否足够具体 | 回到阶段 1，用头脑风暴框架 Q5-Q7 检查 |
| 写着写着发现文献不够 | 停下来做文献补充，不要跳过继续写 |
| 某个论点没有数据支撑 | 标记为 T3，注明需要什么数据，继续推进 |
| 字数超出预算 | 先看哪部分可以压缩（通常是文献综述和讨论） |
| 不确定某个表述是否学术 | 查 3 篇同领域已发表论文，看他们怎么表述 |
| AI 生成的内容不确定是否准确 | 标记为 T3/T4，留给用户验证 |
| 不知道该先写哪部分 | 推荐顺序：方法→结果→讨论→综述→引言→摘要 |

---

## 决策树：遇到写作困难时

```
卡住了？
├─ 不知道写什么 → 回到框架搭建（阶段 2）
│   └─ 框架也不清楚 → 回到选题定位（阶段 1）
├─ 有想法但写不出来 → 用口述或大纲形式先写要点
│   └─ 要点也列不出 → 用头脑风暴框架自问自答
├─ 写了但觉得不好 → 先完成再完善，不要反复改同一段
├─ 数据不够支撑论点 → 标记为 T3，继续写其他部分
└─ 格式/引用太麻烦 → 留到阶段 4 统一处理
```
